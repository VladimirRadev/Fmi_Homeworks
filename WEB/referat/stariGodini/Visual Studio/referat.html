Visual Studio инструменти за уеб разработка

1. Какво e Visual Studio

Visual Studio предоставя мощна интегрирана среда за писане на код, компилиране, изпълнение, дебъгване (както за високо така и за машинно ниво), тестване на приложения, дизайн на потребителски интерфейс (форми, диалози, уеб страници, визуални контроли и други), моделиране на данни, моделиране на класове, изпълнение на тестове, пакетиране на приложения и стотици други функции. Могат да се добавят и плъгини, които повишават функционалността на почти всяко ниво. Visual Studio поддържа различни езици за програмиране (например: C#, VB.NET, C/C++, F#, XML/XSLT, HTML/XHTML, JavaScript, CSS и други) и различни технологии за разработка на софтуер (Win32, COM, ASP.NET, ADO.NET Entity Framework, Windows Forms, WPF, Silverlight и още десетки други Windows и .NET технологии). Съществуват и отделни езикови версии на Visual Studio, които обаче предоставят по-ограничени услуги за потребителя: Microsoft Visual Basic, Visual J#, Visual C#, and Visual C++. Microsoft предоставят безплатна версия на Visual Studio - Visual Studio Community 2015, в която могат да се намерят всякакви инструменти за разработка.

2. Visual Studio за уеб разработка

Visual Studio е една от основните платформи, които се използват за уеб разработка, но е и най-"тежката". Тя предоставя голям набор от инстументи, които ни помагат много по лесно и с възможно по - малко грешки да разработим нашите приложения.

2.1. Localhost

Едно от интересните неща за Visual Studio e, че когато стартирате съответното уеб приложение, то не се отваря чрез пътя до файла на съответния компютър, а чрез така нареченият localhost и свободен порт.

2.2. Auto-complite

В Visual Studio са вложени много усилия, за да помагат на свойте потребители да пишат код възможно най-лесно. Средата е разработена така, че постоянно да довършва, оцветява, подчертава и проверя кода които пишем.

3. Visual Studio инструменти за HTML

3.1. ZenCoding

ZenCoding e един по-бърз начин за писане на HTML код. Синтаксисът му е подобен на този на CSS селекторите. Може да се използва при наличието на плъг-ина Web Essentials. Част от ZenCoding е и така нареченият Lorem Ipsum генератор.

Основен синтаксис на ZenCoding

  • # създава id атрибут
  • . създава class атрибут
  • [ ] създава custom attribute
  • > създава поделемент
  • + създава елемент на същото ниво
  • ^ качва се стъпка нагоре в ДОМ дървото
  • * създава толкова еднакви елемента, колкото сме му задали
  • $ заменя се с поредни числа, започвайки от 1
  • { } създава текст в елемент
Повече информация за ZenCoding!

3.2. Commands

Във Visual Studio са внедрени много съкратени команди за по-лесно писане на код. С командата Shift+Alt+W при селектирането на един таг, той бива обграден с друг. Когато курсора се намира в клас на атрибут, при натискането на Shift+F12 Visual Studio претърсва всички CSS, LESS и SASS за съответния клас. А при натискането на F12 курсорът се премества в CSS, LESS или SASS файла. Visual Studio притежава и специални мета-тагове за Apple iOS, Twitter Cards, Facebook/OpenGraph, Windows 8, Internet Explorer 9+, Viewport

4. Visual Studio инструменти за CSS

4.1. Intellisense

Visual Studio предоставя много и интересни инструменти за улесняване работата на програмиста. Когато в кода е изпуснат vender или property Visual Studio подчертава кода, показва предупреждение и дава възможност за лесно дабявяне на съответния липсващ елемент.

Visual Studio позволява лесна редакция на елемента с еднакви свойства, като използва различни методи за синхронизация. По този начин се избягва допускането на грешки при редактиране на кода.

4.2. Visual Clues

Visual Studio съдържа всички видове цветови гами, включително и тези в CSS3. При всеки избор на цвят, се показва малък квадрат запълнен в съответния цвят. Освен, че показва, как реално изглеждат цветовете, Visual Studio ни позволява да видим как би изглеждал даден шрифт. Когато изберем даден шрифт и сложим мишката върху него, се показват иконите на браузърите, които го поддържат. Някой селектори не се поддържат във всички браузъри, за това само с няколко клика на мишката можем да добавим видимост за Internet Explorer 6,7,8 и 9.

5. Visual Studio инструменти за JavaScript

5.1.Features

Един от интересните и полезни инструменти, които предлага Visual Studio за по-удобно писан е на JavaScript код е JSHint. JSHint е един наистина добър начин да следим дали нашият код спазва определени стандарти и добри практики. Първоначално в неговите опции са добавени базови свойства, но могат да се добавят и допълнителни.

На картинката е показан прозорецът за грешки на Visual Studio. В него получаваме известия за грешки, докато пишем кода, то ест не трябва допълнително да казваме на .js файла да използва JSHint. Ако не искаме да получаваме постоянни известия за грешки от JSHint, може да използваме един от двата файла .jshintrc и .jshintignore за даго игнорираме.

Друга интересна функционалност включена в Visual Studio е разделянето на кода на региони. Тази функционалност за първи път е вклюеччена в пакета Web Essentials 2010, след което добива популярност и продължава да съществува и в слядващите версии.

5.2. Intellisense

  • getElementById - търси #ID референция във всички .css файлове.
  • getElementsByTagName - намира всички HTML тагове с даденото име.
  • getElementsByClassName - търси класове от вида - .classnames във всички .css файлове.
  • NodeJS modules - Visual Studio има функции за по-лесно използване на NodeJS модулите.

5.3. EcmaScript 6

Visual Studio поддържа широка гама от функционалностите на EcmaScript 6, като основните са: classes, let, const, fat arrows and template strings

5.4. Unit testing

Chutzpah (hutz·pah) e разширение на Visual Studio, което помага за по-лесна интеграция на JavaScript юнит тестове. Интересното в случая е, че не е нужно да напускаме средата, за да направим лесно юнит тестове. Чрез десен клик на мишката виждаме функцията "Run JS Tests". Резултатте които получаваме от тестовете се показат в Error list в Output прозореца.
